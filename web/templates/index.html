关于 DMR-ML

一句话概括

DMR-ML = DMR（双重动量轮动）+ ML（机器学习风险门禁）。前半段负责决定"买什么"，后半段负责判断"现在该不该买"。

什么是双重动量

"动量"的意思很朴素：过去一段时间涨得好的资产，短期内继续涨的概率往往更高。这是学术界研究较为充分的市场异象之一，我们的策略正是基于这一规律构建的。

我们同时跟踪沪深300和中证1000这两个宽基指数的动量，哪个更强就持有哪个，这就是"轮动"。如果两个都在跌，说明市场整体走弱，那就都不持有，转为现金等待。

选择宽基指数而不是个股，有几个考虑。一是操作简便，对应的ETF和指数基金都非常成熟，流动性好、费率低、任何人都可以直接买卖。二是分散了个股风险，我们赚的是市场整体的Beta收益（跟随市场涨跌的那部分），然后通过择时来决定什么时候参与、什么时候离场。本质上这是一个择时策略，不做选股。

ML风险门禁

机器学习是一类让计算机从大量历史数据中自动提取规律的方法。与传统的手动编写规则不同，我们将过去的市场数据输入模型，由模型自行学习哪些特征组合往往预示着下行风险。

光靠动量有时会在市场剧烈转向时反应不够快，所以我们加了这一层ML模型作为门禁。它会根据近期的市场特征（波动率、成交量变化、技术指标等），估算未来出现大幅下跌的概率。如果这个概率超过了设定的阈值，无论动量信号怎么说，都强制清仓避险。可以把它理解成一个额外的安全阀，动量说"可以买"，但风控说"现在太危险"，那就不买。

在回测中对比同参数下开启和关闭ML模块的表现，加入ML之后年化收益提升了约3-5%，最大回撤降低了约6%，夏普比率也有明显改善。不过客观来看，改善的主要来源不是"赚得更多"，而是"亏得更少"。风险门禁在几次大幅下跌前提前触发了空仓信号，少亏的部分积累起来就变成了超额收益。

关于阈值，我们用的是双阈值迟滞机制，触发线40%、解除线33%。为什么不是一条线？因为如果只设一条线，概率在临界值附近反复波动时，策略就会频繁进出，既增加了手续费成本，也增加了操作难度。两条线之间留一段缓冲区，只有风险概率真正降到足够低才恢复交易。

为什么选随机森林

随机森林是一种常见的机器学习模型。它会生成很多棵"决策树"，每棵树用略有不同的数据子集和特征子集来做判断，最后把所有树的结果汇总投票，少数服从多数。这种方式让单棵树的判断失误不容易影响整体结论，稳定性比较好。

在搭建ML模块的过程中，我们比较了逻辑回归、SVM、XGBoost、LightGBM、随机森林等多种模型，最终发现随机森林在我们的场景下表现最稳定。它对过拟合的抵抗力较好，不需要太精细的调参，在不同市场环境下的泛化能力比较一致。不是说其他模型不行，只是随机森林在稳定性上的优势最明显。

当前配置是100棵决策树、最大深度限制为5层。深度限制意味着每棵树不会学得太细，这是刻意为之。模型越复杂，越容易把历史数据中的噪声也当成规律来记忆，也就是过拟合，表现为回测结果很好看，但面对新数据时效果明显下降。我们宁可模型简单一些，也希望它在实际运行中的表现和回测中看到的尽量一致。

Purged Walk-Forward 训练

这是模型的训练方式。我们不会拿未来的数据去训练模型然后回头"预测"过去，那样的回测结果没有参考价值。Purged Walk-Forward 的做法是按时间顺序滚动：用过去的数据训练，只预测紧接着的未来，然后窗口往前滑，重复这个过程。"Purged"是指在训练集和测试集之间还额外删掉一段数据，防止标签泄露（因为我们的标签是用未来收益算出来的，如果训练集末尾和测试集开头离得太近，信息会渗透过去）。这套流程比较严格，目的是让回测结果尽可能接近真实世界的表现。

交易成本模拟

回测中我们尽量贴合实际交易去模拟成本。目前的设定是：每次买入或卖出扣除0.03%的手续费。如果是从一个指数直接切换到另一个指数（比如沪深300换到中证1000），相当于先卖后买，手续费按两笔计算共0.06%。持有现金期间按年化3%的无风险利率计入收益。ETF交易目前免征印花税，所以回测中没有单独模拟这一项。宽基ETF的日均成交量通常在数十亿以上，流动性很充裕，实际买卖时的价格偏差（滑点）极小，回测中暂未额外扣除。整体而言，这套成本假设是比较保守和贴近现实的。

关于回测

回测就是用历史数据模拟策略运行的过程，假设从某一天开始执行这套规则，到今天为止会是什么结果。它的价值在于验证策略逻辑是否在过去站得住脚，但要注意：过去有效不等于未来一定有效，回测结果天然会比实盘乐观一些。

起始时间选在2019年1月，主要有两个原因。一是中证1000指数在2018年底之前的数据质量和可投资性都不太理想；二是2019年至今的市场经历了牛市、熊市、震荡市，样本覆盖得比较全面。更早的数据当然可以跑，但参考价值会打折扣。

实际怎么操作

策略每天收盘后计算一次信号，在下一个交易日开盘前通过邮件推送。信号只有两种：买入（持有某个指数）或卖出（转为现金）。收到信号后，在开盘时买入或卖出对应的指数基金即可。沪深300对应的ETF很多（比如510300、159919），中证1000同理（比如560010、159845），选费率低、规模大的就好。

我们的策略是全仓切换：信号说买沪深300就全部仓位买沪深300，说切到中证1000就全部换过去，说卖出就全部转为现金。在开发过程中，我们也尝试过半仓、三分之一仓、五分之一仓等分批调仓的方案，但回测结果显示分仓反而不如全仓切换。可能的原因是，动量策略的核心逻辑就是"强者恒强"，分散仓位等于在确信的方向上打了折扣，稀释了信号的有效性。全仓切换的另一个好处是操作极其简单，信号明确、没有模糊地带，不需要纠结"这次该调多少"，也有效降低了执行时的心理负担。

有一点值得提前说明：趋势跟踪策略天然有一定的滞后性。信号需要等趋势形成后才会触发，这意味着买入时往往不是最低点，卖出时也不是最高点。这是动量策略结构上的特点，不是可以通过调参消除的缺陷。如果把这个模型理解为一个下行风险过滤器，而非追求极致收益的进攻型系统，使用起来会更顺手一些。

你不必完全照做

以上是策略本身的运行方式，但你完全可以根据自己的判断灵活使用。比如信号显示持有中证1000，你可以把它理解为"当前小盘股相对更强"，作为自己选股或配置的参考；信号转为现金避险，你可以据此降低仓位而不必清仓。这个工具提供的是一个视角，最终如何决策，还是取决于你自己的判断。

坦白说，这个策略倡导的是一种相对保守、纪律优先的思路：在不确定的时候宁可少赚，也不冒额外的风险。这是我个人在搭建它时的出发点，但每个人对风险和收益的权衡都不一样，这里提供的只是一个可供参考的框架。

致使用者

你好，感谢你打开这个页面。

DMR-ML 是我在学习量化投资过程中做的一个小项目。它的核心思路并不复杂——用双重动量在沪深300和中证1000之间轮动，再用一层机器学习模型做风险门禁。选这两个宽基指数，是因为它们有对应的ETF和指数基金，任何人都可以低成本地跟随操作。

坦率地说，这个策略并不完美。如果你翻看第五页的逐笔交易记录，会发现它并非每次都能精准地买在谷底、卖在峰顶——趋势跟踪策略天然会有一定的滞后性，有时信号确认时，最佳入场点已经过去了。这是动量策略的固有局限，我不想回避这一点。

但它确实有做得不错的地方。从2019年至今的回测来看，策略最大的优势在于熊市中的避险能力。ML风险门禁在多次大幅下跌前触发了空仓信号——在A股这样的市场里，不亏本身就是一种超额收益。回测中相当一部分累计收益，其实来自"别人在跌的时候我们没有在场"。我个人认为，这个模型更适合被理解为一个下行风险过滤器，而非一个追求极致进攻的交易系统。

关于未来，有一些我想做但尚未实现的事情：接入更多标的（行业ETF、贵金属等避险资产），设计真正不同策略逻辑的模型组合——目前 Adagio（柔板）和 Presto（急板）只是参数不同，底层策略是一致的，我希望将来能有"不同乐章"。也在考虑更及时的信号推送方式。

整个项目已经开源在 GitHub 上。如果你有兴趣，完全可以 fork 一份，自行调整参数，或者搭配 OpenClaw 接入微信、飞书以及 iMessage 等平台做每日推送，让信号自动送达手边，在此基础上构建你自己的策略。用这个网站的邮件订阅和参数面板当然也可以，但我更希望它是一个起点，而不是终点。

最后想说的是，任何模型都只是辅助判断的工具，不能替代你自己的思考。策略倡导的是一种长期主义——坚定地执行纪律，接受短期的不完美。但在真实的投资决策中，每个人都有自己的处境和判断，模型仅供参考。

如果这个小工具对你有哪怕一点点帮助，或者激发了你对量化投资的兴趣，那就是我做这件事最大的回报了。
